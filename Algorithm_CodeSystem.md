# 문자열

## 코드체계

- 문자에 대응하는 숫자를 정한 것
- 전 세계의 모든 문자를 컴퓨터가 일관되게 표현하고 처리하는 것
- ASCII 문자 인코딩 표준 제정

---

### ASCII

- 7-bit 인코딩으로 128문자를 표현
- 33개의 출력 불가능한 제어 문자들과 공백을 비롯한 95개의 출력 가능한 문자들로 이루어

---

## 확장 아스키 (Extended ASCII)

- 표준 문자 이외의 악센트 문자, 도형 문자, 특수 문자, 특수 기호 등 부가적인 문자를 128개 추가
- 8-bit를 모두 사용함으로써 추가적인 문자 표현 가능
- 프로그램이나 컴퓨터 또는 프린터가 그것을 해독할 수 있도록 설계되어 있어야만 올바르게 해독 가능

---

## 유니코드

- 다국어 처리를 위한 표준 코드 체계
- 이모지도 유니코드 문자
- UCS-2 / UCS-4 → 유니코드를 저장하는 변수의 크기를 정의함

---

## 바이트 단위 저장 순서

- 순서가 정해지지 않은 경우 잘못된 해석 가능성
- Endian : 여러 바이트로 이루어진 데이터를 저장하는 방식
- Big-endian : 상위 바이트(MSB)를 가장 낮은 주소에 저장
- Little-endian : 하위 바이트(LSB)를 가장 낮은 주소에 저장

---

## 유니코드 인코딩 (UTF - Unicode Transformation Format)

- UTF - 8 : web
    - Min : 8-bit
    - Max : 32-bit (1 Byte * 4)
- UTF-16 : windows, java
    - Min : 16-Bit
    - Max : 32-bit (2 Byte * 2)
- UTF-32 : unix
    - Min : 32-bit
    - Max : 32-bit (4 Byte * 1)

---

### PyCharm에서 인코딩 방식을 확인하는 방법

- 인코딩과 상관없지만 OS에 따라 줄 바꿈을 표시하는 문자(코드)가 다름
- 윈도우는 CR(13)과 LF(10) 두 글자를 사용
- Unix와 macOS는 LF(10) 하나 글자를 사용
