# Algorithm

# 알고리즘

- 유한한 단계를 통해 문제를 해결하기 위한 절차나 방법
- 컴퓨터가 어떤 일을 수행하기 위한 단계적 방법

---

## 알고리즘의 효율

- 공간적 효율성과 시간적 효율성
- 공간적 효율성 : 알고리즘이 필요로 하는 메모리 공간
- 시간적 효율성 : 알고리즘이 작업을 완료하는 데 걸리는 시간 (입력 크기 n에 대한 연산 횟수)
- 효율성 ↔ 복잡도
- 복잡도가 높을수록 효율성이 저하됨

---

# 복잡도의 점근적 표기

- 시간 복잡도는 입력 크기에 대한 함수로 표기
- 입력 크기 n이 무한대로 커질 때의 복잡도를 간단히 표현하기 위해 사용하는 표기법
    - O(BIG-O) 표기

---

## O(Big-Oh) 표기

- Big-Oh 표기는 복잡도의 점근적 상한을 나타냄
- 복잡도가 f(n) = 2*n^2 - 7*n + 4 라면 O-표기는 O(n^2)

---

### 자주 사용하는 O-표기

1. O(1) : 상수 시간
2. O(logn) : 로그(대수) 시간
3. O(n) : 선형 시간
4. O(nlogn) : 로그 선형 시간
5. O(n^2) : 제곱 시간
6. O(n^3) : 세제곱 시간

---

## 복잡도 비교

- O(logN) 은 O(1)보다는 느리지만, N이 작은 경우 유사한 성능
- O(NlogN) 은 O(N) 보다는 느리지만, N이 작은 경우 유사한 성능
